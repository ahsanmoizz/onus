Why Onus?

AI agents run with broad capabilities. Onus gives you control.

Real-time Interception

Every tool call is intercepted and evaluated against policy before execution. Block file access, network writes, and credential leaks in real time.

onus run "rm -rf /data" → DENIED

Human Approval

Dangerous actions require explicit human approval. Built-in approval server with local dashboard.

onus approvals serve → http://127.0.0.1:9191

Signed Policies

Cryptographically sign your safety policies with Ed25519. Verifiable, tamper-proof rule enforcement.

onus rules sign policy.json → policy.signed.json

Immutable Audit

Every action is logged with hash-chaining. Verify integrity end-to-end. Tampering is instantly detectable.

onus verify → Chain integrity: VALID

Secret Redaction

Content-aware detection of JWTs, connection strings, private keys, and high-entropy secrets. They never appear in logs.

token: "eyJhbGci..." → [REDACTED]

Container Isolation

Run agents in sandboxed workspaces with filesystem, network, and process containment (L3).

onus workspace create --sandbox

Enforcement Levels

Four tiers of protection, from best-effort to full containment

L1 — Cooperative

Best-effort hook

Agent-side hooks intercept tool calls. The agent can opt out. Easy to set up, minimal overhead.

L2 — Intercepted

Bootstrap enforcement

Onus sits between the agent and tools. Action interception is mandatory for routed operations. Requires MCP proxy or shell wrapper.

L3 — Isolated

Containerized execution

Agent runs inside a sandbox with filesystem, network, and process isolation. Onus controls the boundary.

L4 — Sovereign

Full authority control

Onus holds all credentials and authority. Agents request actions; Onus executes them. No direct access to sensitive resources.
