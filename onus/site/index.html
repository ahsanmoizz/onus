# Onus - AI Agent Firewall Technical alpha safety layer for actions explicitly routed through Onus. It evaluates proposed actions against policy rules and blocks dangerous operations on supported paths before they execute. ## Features - **Pre-action checks on routed paths** - evaluates supported actions before execution when the agent/tool uses an Onus integration - **Policy engine** - 13 built-in safety and scope rules, fully configurable TOML - **Prototype integrations** - Claude Code hook adapter, experimental MCP proxy, shell wrapper, Python SDK, VS Code extension - **Audit trail** - local SQLite audit records with SHA-256 hash-chain verification - **Goal drift detection** - validates file touches against declared scope where scope tracking is wired in - **Tier 2 heuristics** - catch dangerous patterns by magnitude analysis - **Low latency** - local Rust binary, no cloud dependency for deterministic evaluation ## Quick Start ```bash # Install (Linux/macOS) curl -fsSL https://github.com/Gitlawb/onus/releases/latest/download/install.sh | bash # Install (Windows PowerShell) iex "& { $(irm https://github.com/Gitlawb/onus/releases/latest/download/install.ps1) }" # Install (npm) npx onus ``` ## Documentation This page is informational, not the live Onus dashboard. For the current runtime state, run `onus --help`, `onus dashboard`, and the repository current-state reports. ## Architecture ```text Agent Action -> Onus IPC -> Policy Engine -> Audit Log -> Verdict -> Block/Allow/Correct | Tier 1 Rules Tier 2 Heuristics ``` ## License MIT (c) 2026 Onus
