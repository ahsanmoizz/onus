# Claude Code L1 Runtime Report

Date: 2026-06-15

## Claim Boundary

Claude Code integration is an L1 cooperative hook integration and must remain
labeled:

```text
BEST-EFFORT
```

Onus only participates when Claude Code loads and runs the configured
`PreToolUse` hook. Direct tool execution, disabled hooks, safe mode, bare mode,
misconfigured settings, unsupported Claude tool events, or hook process failure
can bypass or degrade this integration. This is not L2, L3, or L4 enforcement.

## Hook Configuration

Installer configuration now writes a documented `PreToolUse` hook shape:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "<path-to-onus>",
            "args": ["claude-hook"],
            "timeout": 5,
            "statusMessage": "Onus reviewing Claude Code tool call"
          }
        ]
      }
    ]
  }
}
```

The hook command emits Claude Code hook output with
`hookSpecificOutput.permissionDecision` set to `allow`, `deny`, or `ask`.
